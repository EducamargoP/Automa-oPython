
Projetos de Automação, Banco de Dados e Segurança

Este repositório reúne três projetos práticos que serão publicados ao longo de 30 dias, com foco em subir 1 conteúdo por dia. A ideia é mostrar evolução contínua e prática em Python, Linux, PostgreSQL e Segurança da Informação.

📅 Plano de Publicação

Cada dia será dedicado a subir um conteúdo novo (código, README ou exemplo).

Em 30 dias, o repositório terá uma coleção completa de automação, banco de dados e segurança.

Os três projetos principais serão publicados em sequência, cada um com incrementos diários.
-----------------------------------------------------------------------------------------------------
🐍 Projeto 1 – Automação em Python (nível iniciante/intermediário)

Título: Gerenciador de tarefas automatizado

Objetivo

Criar um script em Python que lê uma lista de tarefas de um arquivo .csv ou banco de dados (MySQL/PostgreSQL) e automatiza notificações.

Funcionalidades

Leitura de tarefas de um arquivo CSV.

Verificação de tarefas próximas do vencimento (usando datetime).

Exibição de notificações no terminal.

Possibilidade futura de envio de e-mails automáticos (usando smtplib).

Tecnologias usadas

Python (csv, datetime, os, smtplib).

Impacto

Demonstra lógica de programação, manipulação de dados e automação de processos simples.
--------------------------------------------------------------------------------------------------
🐧 Projeto 2 – Linux + PostgreSQL (nível intermediário)

Título: CRUD de usuários em PostgreSQL com interface CLI

Objetivo

Criar um sistema simples de cadastro, leitura, atualização e exclusão de usuários, rodando em ambiente Linux.

Funcionalidades

Inserir novos usuários no banco.

Listar usuários cadastrados.

Atualizar informações de usuários.

Excluir registros.

Interface de linha de comando (CLI) para interação.

Tecnologias usadas

Python com biblioteca psycopg2.

PostgreSQL.

Linux para execução e administração.

Impacto

Mostra domínio de banco de dados, integração com Python e prática em Linux.
--------------------------------------------------------------------------------------------------
🛡️ Projeto 3 – Segurança da Informação / SOC (nível intermediário/avançado)

Título: Simulador de monitoramento de logs

Objetivo

Criar um script em Python que lê arquivos de log (ex.: Apache ou Nginx) e detecta padrões suspeitos.

Funcionalidades

Leitura de arquivos de log.

Identificação de múltiplas tentativas de login.

Detecção de acessos de IPs diferentes em curto intervalo.

Geração de alertas simples (salvar em arquivo ou enviar e-mail).

Tecnologias usadas

Python (manipulação de arquivos e strings).

Conceitos de Segurança da Informação.

Impacto

Demonstra aplicação prática de Segurança da Informação, lógica de detecção e monitoramento de eventos.
-----------------------------------------------------------------------------------------------------
🚀 Como cada projeto funciona

Projeto 1: Executar o script em Python para ler o CSV e mostrar notificações de tarefas próximas do vencimento.

Projeto 2: Rodar comandos em Linux para interagir com o banco PostgreSQL via Python (CRUD completo).

Projeto 3: Executar o script em Python apontando para um arquivo de log e receber alertas de atividades suspeitas.

📌 Conclusão

Este repositório será atualizado diariamente durante 30 dias, com foco em evolução prática e consistente. Cada projeto mostra uma etapa diferente: do iniciante em automação, passando por banco de dados em Linux, até aplicações de segurança da informação.
